# 後処理

作成・利用したリソースを削除します。

利用予定の無いリソースは削除をお勧めします。

## AWS リソース

### Amplify Sandbox

Sandbox 環境を削除します。

削除方法がいくつか存在しますので、いずれかをお試しください。

#### `npx ampx sandbox` を実行した CLI 環境が残っている場合

`npx ampx sandbox` から削除する方法です。

```shell
npx ampx sandbox delete
? Are you sure you want to delete all the resources in your sandbox environment (This can't be undone)? (y/N) > y
```

`[Sandbox] Finished deleting.` と表示されれば完了です。

#### ホスティングされたページが残っている場合

Amplify コンソール画面にてスタックを削除する方法です。

1. AWS マネージメントコンソールにて Amplify を開く
2. 「サンドボックを管理」をクリック
3. サンドボックス名が `amplify-gen2workshop` から始まるものを選択
4. 「選択項目を削除」をクリック
5. 「削除」と入力し、確認をクリック

一覧に表示されなくなれば完了です。

#### ホスティングされたページが残っていない場合

ホスティングされたページが存在しない状態の場合、Amplify Sandbox を管理するコンソール画面に到達することができません。

なので、CloudFormation コンソール画面にてスタックを直接削除する方法です。

1. AWS マネージメントコンソールにて CloudFormation を開く
2. スタック名が `amplify-gen2workshop` から始まるもののうち、`ネストされた` とラベルされていないスタックを選択
3. 「削除」をクリック

更新ボタンをクリックして、スタック一覧から表示されなくなれば完了です。

### Amplify ホスティング

ホスティングした環境を削除します。

1. AWS マネージメントコンソールにて Amplify を開く
2. 「amplify-gen2-workshop」と表示されているアプリをクリック
3. 「アプリケーションの設定」の「全般設定」をクリック
4. 「アプリの削除」をクリック
5. 「削除」と入力し、削除をクリック

## GitHub リソース

### Codespaces

Codespaces 環境を削除します。

1. 作成したリポジトリのページで「Code」ボタンをクリック
2. 「Codespaces」タブを選択
3. `Active` と表示されている右側の 3 点リーダーをクリック
4. 「Delete」を選択
